How to read this deck

How the seed remembers.

A seed agent does not keep its memory in the chat window — chat is capped and lost at compaction. Instead it remembers on a bus: files literally named CLAUDE.md that are its working memory. This deck walks that substrate from the top down — the Markov self-model the seed reads at session start, the working-memory bus and the teeth that updating it grows, the layered hierarchy (root injected always, local surfaced on demand), the four-state lifecycle that inflates then deflates, the four footer anchors that make a file reachable, the three durable surfaces the bus condenses into, and the soft size caps plus the forgetting discipline that moves overflow rather than dropping it. Each card is one focused idea, with a clean diagram you can hover and click.
