How to read this deck

How the seed splits its work.

A seed agent cannot hold a large job in one context window, so it divides cognitive labour: the main session keeps the cross-cutting judgment (~20%), and dispatched subagents carry the parallel bulk (~80%). This deck walks how that split is mechanized: the direct-action budget that turns "delegate more" into a counted resource, the gate exemption that lets fan-out run unthrottled, the per-plugin pools and per-phase rosters the labour is dispatched into, the two shared wiring hooks that discover and account for every subagent, and the richer optional form — the Workflow execution layer and the built, instruction-borne fractal inner cycle. Each card is one focused idea, with a clean diagram you can hover and click.
